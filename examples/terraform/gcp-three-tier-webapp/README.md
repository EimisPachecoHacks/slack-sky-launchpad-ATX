# GCP Three-Tier Web Application

Production-ready Terraform configuration for deploying a three-tier web application on Google Cloud Platform, generated by **Skyrchitect**.

## Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                     Cloud CDN + HTTPS LB                    │
│                    (Global Load Balancer)                    │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────▼──────────────────────────────────┐
│               Managed Instance Group (Web/App)              │
│            e2-medium × 1-5 (autoscaling on CPU)             │
│                  Private subnet (10.0.1.0/24)               │
└─────────┬───────────────────────────────────┬───────────────┘
          │                                   │
┌─────────▼──────────┐            ┌───────────▼───────────────┐
│   Cloud SQL        │            │   Memorystore Redis       │
│   PostgreSQL 15    │            │   (Session/Cache tier)    │
│   Regional HA      │            │   Private Service Access  │
│   Private IP only  │            └───────────────────────────┘
│   SSL enforced     │
└────────────────────┘
          │
┌─────────▼──────────┐
│   Cloud Storage    │
│   Static Assets    │
│   CDN-backed       │
└────────────────────┘
```

## Services Provisioned

| Service | Purpose | Notes |
|---|---|---|
| **VPC Network** | Custom network with private subnets | Deny-all default, explicit allow rules |
| **Cloud NAT** | Outbound internet for private instances | No public IPs on compute |
| **Managed Instance Group** | Web/app tier with autoscaling (1-5) | Health-checked, rolling updates |
| **HTTPS Load Balancer** | Global L7 load balancer with CDN | SSL termination, URL map |
| **Cloud SQL PostgreSQL 15** | Relational database tier | Private IP, HA, automated backups, SSL |
| **Memorystore Redis** | In-memory cache / session store | Private service access |
| **Cloud Storage** | Static asset hosting | Versioning, lifecycle rules, CDN |
| **Cloud Armor** *(optional)* | WAF / DDoS protection | Extend via security policy |

## Prerequisites

1. **GCP Project** with billing enabled
2. **Terraform** >= 1.5.0 installed
3. **gcloud CLI** authenticated (`gcloud auth application-default login`)
4. **APIs** enabled (handled automatically by the `google_project_service` resources):
   - Compute Engine
   - Cloud SQL Admin
   - Redis
   - Service Networking
   - Cloud Resource Manager
5. **GCS bucket** for Terraform state (update `providers.tf` backend block)
6. **Permissions**: `roles/editor` or scoped roles for compute, SQL, redis, networking

## Usage

```bash
# 1. Clone and navigate
cd examples/terraform/gcp-three-tier-webapp/

# 2. Create your variable file
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your project-specific values

# 3. Update the backend bucket in providers.tf
#    Replace REPLACE_WITH_YOUR_STATE_BUCKET with your GCS bucket name

# 4. Initialize Terraform
terraform init

# 5. Review the execution plan
terraform plan

# 6. Apply the configuration
terraform apply

# 7. Retrieve outputs
terraform output load_balancer_ip
terraform output -json database_connection_name
```

## Security Features

- **No public IPs** on compute instances; all egress via Cloud NAT
- **Deny-all default firewall** with explicit allow rules only
- **IAP SSH access** instead of exposing port 22 to the internet
- **Private IP databases** with no public endpoint
- **SSL/TLS enforced** on all database connections
- **Dedicated service accounts** per workload (no default compute SA)
- **Deletion protection** enabled on stateful resources in production
- **Versioned Cloud Storage** with lifecycle policies
- **VPC Service Controls** ready (extend as needed)

## Customization

- **Startup script**: Replace the placeholder in the instance template's `metadata_startup_script`
- **SSL certificate**: Add a managed certificate via `google_compute_managed_ssl_certificate`
- **Cloud Armor**: Attach a security policy to the backend service for WAF/DDoS
- **Monitoring**: Add `google_monitoring_alert_policy` resources for uptime and latency
- **DNS**: Configure `google_dns_record_set` to point your domain to the load balancer IP

## Cleanup

```bash
# Disable deletion protection first (if enabled)
terraform apply -var="enable_deletion_protection=false"

# Destroy all resources
terraform destroy
```
